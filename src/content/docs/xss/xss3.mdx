## DOM XSS in document.write sink using source location.search

<div style="text-align: justify;">
  This lab demonstrates a DOM-based cross-site scripting (XSS) vulnerability using the `document.write` function. The search query tracking functionality takes user input from `location.search` and writes it to the page without sanitization. This allows an attacker to inject arbitrary JavaScript into the page.
</div>

<Aside>
  Start Burp Suite before running the script â€” it uses a proxy for visibility.
</Aside>

### How Exploit Works
- The search box input is reflected via `document.write`.
- Inspecting the page shows the input placed inside an `img src` attribute.
- Breaking out of the `img` tag allows arbitrary script execution.
- A simple XSS payload like `"><svg onload=alert(1)>` executes JavaScript via the `onload` event.
